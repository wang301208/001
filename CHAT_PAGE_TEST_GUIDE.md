# ChatPage 快速测试指南

## 🚀 快速开始

### **1. 启动后端网关**

```bash
# Windows PowerShell
$env:ZHUSHOU_GATEWAY_TOKEN="dev-token-123"
node zhushou.mjs gateway --bind lan --port 3000 --allow-unconfigured
```

等待看到：
```
[gateway] ready (5 plugins: acpx, browser, device-pair, phone-control, talk-voice; XX.Xs)
```

### **2. 访问聊天页面**

浏览器打开：**http://localhost:3000/chat**

## 🐛 常见问题排查

### **问题 1：页面一直显示加载遮罩**

**症状**：
- 打开页面后始终显示加载动画
- 超过5秒仍未显示内容

**排查步骤**：
1. 检查后端服务是否运行
   ```bash
   Get-Process node
   ```

2. 检查端口占用
   ```bash
   netstat -ano | findstr :3000
   ```

3. 查看浏览器控制台
   - 按 F12 打开开发者工具
   - 切换到 Console 标签
   - 查看是否有红色错误信息

**解决方法**：
- 重启后端服务
- 检查防火墙设置
- 确认环境变量 `ZHUSHOU_GATEWAY_TOKEN` 已设置

---

### **问题 2：发送消息后无响应**

**症状**：
- 用户消息显示后
- "思考中..."指示器一直存在
- 超过10秒仍无AI响应

**排查步骤**：
1. 检查后端日志
   ```bash
   cat C:\Users\30676\.zhushou\logs\gateway.log
   ```

2. 查看浏览器 Network 标签
   - 按 F12 → Network
   - 查找失败的 API 请求
   - 查看响应状态码和错误信息

3. 验证模型配置
   ```bash
   # 检查是否有可用的AI模型
   zhushou agents list
   ```

**解决方法**：
- 确认AI模型已正确配置
- 检查API密钥是否有效
- 查看后端日志中的具体错误

---

### **问题 3：历史消息未加载**

**症状**：
- 刷新页面后
- 之前的对话丢失
- 只显示欢迎消息

**排查步骤**：
1. 检查会话是否存在
   ```bash
   curl http://localhost:3000/api/sessions
   ```

2. 检查消息是否保存
   ```bash
   curl http://localhost:3000/api/sessions/{sessionId}/messages
   ```

**解决方法**：
- 确认后端数据库正常工作
- 检查会话ID是否正确传递
- 查看前端控制台是否有解析错误

---

### **问题 4：离线模式无法退出**

**症状**：
- 显示"离线模式"
- 点击"重新连接"无效
- 仍然无法发送消息

**排查步骤**：
1. 确认后端服务已启动
2. 检查网络连接
3. 查看浏览器控制台错误

**解决方法**：
- 完全刷新页面（Ctrl+F5）
- 清除浏览器缓存
- 重启后端服务后重新访问
