# 前端界面优化 - 快速使用指南

## 🚀 快速开始

### **1. 启动后端网关**

```bash
# Windows PowerShell
$env:ZHUSHOU_GATEWAY_TOKEN="dev-token-123"
node zhushou.mjs gateway --bind lan --port 3000 --allow-unconfigured

# Linux/Mac
export ZHUSHOU_GATEWAY_TOKEN="dev-token-123"
node zhushou.mjs gateway --bind lan --port 3000 --allow-unconfigured
```

等待看到：
```
[gateway] ready (5 plugins: acpx, browser, device-pair, phone-control, talk-voice; XX.Xs)
```

### **2. 访问前端页面**

浏览器打开：**http://localhost:3000**

## 📊 DashboardPage（系统仪表盘）

### **功能说明**

系统仪表盘显示4个关键指标和详细统计信息：

#### **顶部统计卡片**

| 卡片 | 说明 | 数据来源 |
|------|------|---------|
| WebSocket 状态 | 实时连接状态 | WebSocket 连接 |
| 渠道活跃度 | 活跃渠道数量/总数 | 渠道列表 API |
| 任务执行 | 运行中/完成/失败任务数 | 任务管理 API |
| 系统状态 | 冻结状态 + 代理/项目数 | 治理状态 API |

#### **详细统计区域**

**左侧：治理层概览**
- 活跃代理数量
- 演化项目数量
- 沙盒实验数量
- 主权边界状态

**右侧：任务执行统计**
- 运行中任务数
- 已完成任务数
- 失败任务数
- 总任务数

### **使用场景**

**场景 1：监控系统健康状态**
1. 查看 WebSocket 状态是否为"已连接"（绿色）
2. 检查系统状态是否为"正常"（绿色）
3. 观察渠道活跃度百分比

**场景 2：跟踪任务执行情况**
1. 查看运行中任务数量
2. 监控失败任务是否有异常增长
3. 对比完成与失败的比例

**场景 3：治理层审计**
1. 检查活跃代理数量是否符合预期
2. 查看演化项目和沙盒实验进度
3. 确认主权边界未被冻结

## 🔌 ChannelsPage（渠道管理）

### **功能说明**

渠道管理页面用于查看和管理消息渠道的连接状态。

#### **主要功能**

1. **查看渠道列表**
   - 渠道名称和 ID
   - 渠道类型（Telegram、Discord 等）
   - 连接状态（已连接/未连接）
   - 最后活动时间

2. **连接/断开渠道**
   - 点击绿色插头图标：连接渠道
   - 点击红色断开图标：断开渠道
   - 操作时按钮显示加载状态

3. **刷新渠道列表**
   - 点击右上角刷新按钮
   - 重新从后端获取最新数据

4. **错误处理**
   - 操作失败时显示红色 Alert
   - 可关闭错误提示
   - 支持重试操作

### **使用场景**

**场景 1：批量连接渠道**
1. 进入渠道管理页面
2. 依次点击每个渠道的绿色插头图标
3. 等待每个操作完成（按钮停止旋转）
4. 确认所有渠道状态变为"已连接"

**场景 2：排查连接问题**
1. 查看哪些渠道显示"未连接"（红色）
2. 尝试重新连接失败的渠道
3. 如果仍然失败，查看错误提示信息
4. 检查后端日志获取详细原因

**场景 3：定期维护**
1. 点击刷新按钮更新渠道列表
2. 检查最后活动时间是否过期
3. 断开长时间未使用的渠道
4. 记录活跃渠道数量变化

## 🔧 故障排查

### **问题 1：页面一直显示骨架屏**

**症状**：
- 打开页面后始终显示灰色占位块
- 控制台无错误日志

**排查步骤**：
1. 检查后端网关是否正常运行
   ```bash
   # 查看网关进程
   Get-Process node
   
   # 查看端口占用
   netstat -ano | findstr :3000
   ```

2. 检查认证令牌是否正确
   ```bash
   echo $env:ZHUSHOU_GATEWAY_TOKEN
   ```

3. 查看浏览器控制台网络请求
   - 按 F12 打开开发者工具
   - 切换到 Network 标签
   - 刷新页面查看 API 请求状态

**解决方法**：
- 重启网关服务
- 确认环境变量设置正确
- 检查防火墙是否阻止了端口

---

### **问题 2：渠道操作失败**

**症状**：
- 点击连接/断开按钮后显示红色 Alert
- 错误信息："连接渠道失败"

**排查步骤**：
1. 查看后端网关日志
   ```bash
   # 日志位置
   C:\Users\30676\.zhushou\logs\gateway.log
   ```

2. 检查渠道配置是否存在
   ```bash
   # 查看配置文件
   cat ~/.zhushou/channels.json
   ```

3. 测试 API 连通性
   ```bash
   # 使用 curl 测试
   curl http://localhost:3000/api/channels
   ```

**解决方法**：
- 在后端配置正确的渠道信息
- 检查渠道所需的认证密钥
- 重启网关服务使配置生效

---

### **问题 3：WebSocket 连接失败**

**症状**：
- Dashboard 显示"未连接"（红色）
- 无法获取实时数据

**排查步骤**：
1. 检查 WebSocket 端口
   ```bash
   netstat -ano | findstr :3000
   ```

2. 查看浏览器控制台
   ```
   [useGovernanceWebSocket] 连接失败: ...
   ```

3. 检查认证令牌
   ```bash
   # WebSocket 需要相同的令牌
   echo $env:ZHUSHOU_GATEWAY_TOKEN
   ```

**解决方法**：
- 确认网关服务正在运行
- 检查防火墙规则
- 清除浏览器缓存后重试

## 📈 性能监控

### **关键指标**

**页面加载时间**
- 目标：< 2 秒
- 测量方法：浏览器 Performance API
  ```javascript
  performance.getEntriesByType('navigation')[0].loadEventEnd - 
  performance.getEntriesByType('navigation')[0].startTime
  ```

**API 响应时间**
- 目标：< 500ms
- 查看方法：浏览器 Network 标签
- 关注字段：Time / Duration

**渲染性能**
- 目标：60 FPS
- 检测方法：Chrome DevTools Performance 标签
- 避免长任务（Long Tasks > 50ms）

### **优化建议**

1. **启用浏览器缓存**
   - 静态资源设置 Cache-Control
   - API 响应使用 ETag

2. **代码分割**
   - 路由级别的懒加载
   - 大型组件动态导入

3. **图片优化**
   - 使用 WebP 格式
   - 设置合适的尺寸
   - 懒加载非首屏图片

## 📞 技术支持

### **常见问题 FAQ**

**Q: 为什么页面加载很慢？**
A: 首次加载需要获取后端数据，确保网关服务正常运行且网络通畅。

**Q: 如何知道操作是否成功？**
A: 成功的操作会更新 UI 状态（如渠道变为"已连接"），失败会显示红色 Alert。

**Q: 可以同时操作多个渠道吗？**
A: 不可以。系统设计为一次只能操作一个渠道，防止并发冲突。

**Q: 数据多久刷新一次？**
A: 页面挂载时自动加载一次，之后可以手动点击刷新按钮。

**Q: 离线模式下能使用吗？**
A: 部分功能可用，但无法获取最新的后端数据。
