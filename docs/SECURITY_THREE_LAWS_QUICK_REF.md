# 安全三大法则 - 快速参考

## 🎯 核心要点

### 🔒 第一法则：主权边界不可侵犯
**人类保留最终控制权，系统不得自我授权突破主权边界。**

- ✅ 人类专属权力：宪法修正、根权限扩展、高风险网络开放、系统冻结/恢复
- ❌ 禁止行为：自我授权、绕过审计、移除回滚路径
- ⚡ 自动响应：检测到违规 → 立即冻结 → 保存证据 → 上报人类

---

### 🔍 第二法则：所有变更必须可审计可回滚
**没有审计追踪和回滚路径的变更是非法的。**

- ✅ 审计要求：可归因、可追溯、不可篡改、实时记录
- ✅ 回滚要求：回滚计划、回滚测试、回滚触发器、回滚执行
- 📊 风险分级：低（自主）→ 中（沙盒+QA）→ 高（委员会+审计）→ 严重（仅人类）

---

### 🧪 第三法则：实验前验证，证据驱动晋升
**所有变更必须在沙盒宇宙中证明自己，才能晋升到生产环境。**

- ✅ 沙盒要求：隔离环境、真实场景、充分观察、量化证据
- 🔄 晋升流程：提案 → 沙盒实验 → QA 验证 → 审计审查 → 晋升决策 → 晋升后观察
- ❌ 禁止行为：跳过沙盒、伪造证据、绕过 QA、隐瞒失败

---

## ⚡ 快速检查

### 变更前问自己：
1. **这个变更是否侵犯了主权边界？** → 如果是，需要人类批准
2. **这个变更是否可审计可回滚？** → 如果否，不能执行
3. **这个变更是否在沙盒中验证过？** → 如果否，先做实验

### 部署前确认：
- [ ] 提案记录完整
- [ ] 实验报告充分
- [ ] QA 报告通过
- [ ] 审计审查完成
- [ ] 回滚计划测试过
- [ ] 人类批准（如需要）

---

## 🚨 违规处理

| 严重程度 | 示例 | 响应 |
|---------|------|------|
| 信息级 | 本地策略不匹配 | 记录日志，自动修复 |
| 警告级 | 无 QA 晋升 | 冻结子系统，开启事件 |
| 严重级 | 宪法绕过、审计篡改 | **立即冻结 + 人工干预** |
